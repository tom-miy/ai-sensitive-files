# Scope and Safety Checks

このページではプロジェクトの境界を具体的に説明します。
対象となる流れは単純です。
リポジトリに「機密だと分かっている場所」がある。
その場所を AI ignore、gitignore、暗号化/復号計画、
コミット前チェックで同じように扱いたい、という場面です。

対象になりやすい場所:

- `.env`
- `.aws/credentials`
- `.agent-privacy-guard/entities.local.yaml`
- devcontainer 構成で使う `/workspaces/.runtime-secrets/app.env`
- 1Password / Bitwarden からローカルに書き出すファイル

## このツールが検査すること

- ポリシー項目の必須項目が足りない
- 生成済み ignore/config ファイルがポリシーより古い
- `.gitignore` に、ポリシーから生成した項目が反映されていない
- `commit_block` 対象が Git 管理されている
- ポリシー上は残してはいけない復号後の平文ファイルが存在する
- 復号後の平文ファイルが暗号化済みファイルより新しい
- そのため、コミット前に暗号化し直す必要がある
- 暗号化済みファイルがローカルの平文ファイルより新しい
- そのため、編集前に復号し直す必要がある

## 他のツールに任せること

- プロンプトの検査や匿名化はプロンプトゲートウェイの責務
- 危険コマンドのブロックはローカルエージェントフックの責務
- 実行時のファイル読み取りブロックはサンドボックスの責務
- devcontainer やエージェント固有の権限設定の責務でもある
- ソースコード内の未知のシークレット探索は secret scanner の責務
- SOPS/age の鍵はそれぞれのツールの責務
- パスワードマネージャーへのアクセスもそれぞれのツールの責務
- リポジトリアクセスはリポジトリ権限の責務

未知の漏えい検知には secret scanner を使ってください。
秘密値の保存には SOPS/age やパスワードマネージャーを使います。
AI のファイル読み取りを OS やツールのレベルで止めたい場合は、
devcontainer やローカルエージェントフックなどと組み合わせます。
